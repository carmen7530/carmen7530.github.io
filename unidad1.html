Introducción a la Estructura de Datos

Clasificación de las estructuras de datos

Las estructuras de datos pueden ser lineales (e.g., arreglos, listas, pilas y colas) o no lineales (e.g., árboles y grafos). Estas se eligen según la naturaleza del problema y la eficiencia requerida en las operaciones.

Tipos de datos abstractos (TDA)

Los TDA son modelos conceptuales que definen conjuntos de datos y las operaciones permitidas sobre ellos, sin considerar la implementación. Ejemplos incluyen listas, pilas, colas, diccionarios, conjuntos y árboles, los cuales resuelven problemas específicos en el manejo de información.

Ejemplos de TDA

Cada TDA tiene implementaciones específicas según el lenguaje de programación utilizado, como listas con arreglos dinámicos o listas enlazadas, pilas y colas con listas o estructuras dedicadas, y árboles y grafos para representar relaciones jerárquicas o redes.

Manejo de memoria

Este apartado distingue entre memoria estática, donde el tamaño de los datos es fijo y se asigna en tiempo de compilación, y memoria dinámica, que se asigna durante la ejecución, permitiendo mayor flexibilidad al costo de una mayor complejidad en el manejo.

Análisis de algoritmos

El análisis se centra en evaluar dos aspectos principales: la complejidad temporal, que mide cómo el tiempo de ejecución crece con el tamaño de la entrada, y la complejidad espacial, que evalúa la memoria que utiliza el algoritmo. Ambos factores determinan la eficiencia, siendo clave para seleccionar la solución más adecuada en un contexto dado.
